# Qwen3-TTS Ascend 知识站页面模板设计

日期：2026-07-16

## 1. 目的与范围

本规格定义 Qwen3-TTS 到 Ascend 910B 学习站的基础页面视觉、布局、侧栏交互、响应式行为、降级方式和验收标准。它服务于源码走读、迁移映射和长篇技术教学页面，不定义研究内容，也不在本阶段实现正式 `site/` 文件。

主要阅读设备包括 27 英寸左右的宽屏桌面、13–16 英寸笔记本，并保证手机端可以检索和阅读。桌面源码走读体验优先，但任何断点都不能隐藏正文、证据或引用。

## 2. 已选视觉方向

采用用户确认的融合方案：

- 布局取自“工程工作台”：左侧章节树、中央正文、右侧页内目录与证据。
- 配色取自“技术专著”：暖纸张背景、克制的棕红强调色、深色代码块和衬线标题。
- 信息密度保持工程工具级别，不采用大面积装饰图、营销式卡片或占用正文空间的视觉效果。
- 页面运行时不加载远程字体、CSS 或 JavaScript，保证离线阅读。

## 3. 视觉令牌

### 3.1 色彩

| 用途 | 值 | 说明 |
| --- | --- | --- |
| 页面画布 | `#f7f1e7` | 暖纸张底色 |
| 顶部栏 | `#e8ddcd` | 与正文区分但不过度突出 |
| 左侧章节栏 | `#eee5d8` | 稍深于正文 |
| 中央正文 | `#fcf9f3` | 最高亮度阅读区域 |
| 右侧证据栏 | `#f1e8dc` | 与章节栏保持同一色系 |
| 主文字 | `#2b251f` | 深暖灰，保证正文对比度 |
| 次要文字 | `#675a4d` | 元数据、解释和导航 |
| 主强调色 | `#934735` | 当前章节、主要按钮、重要迁移点 |
| 辅助强调色 | `#9b6c29` | 分组标题和警示 |
| 边框 | `#cdbda7` | 栏目和卡片分隔 |
| 代码背景 | `#22211e` | 深暖黑 |
| 代码正文 | `#f3eee4` | 暖白 |

证据标签使用低饱和底色和深色文字：绿色表示“已证实”，棕红表示 Ascend 相关，金色表示“待真机验证”。颜色不作为唯一状态信号，标签必须同时包含文字。

### 3.2 字体与排版

- UI、正文和表格使用系统无衬线字体栈，避免远程依赖并保证中文覆盖。
- 一级、二级标题使用系统衬线字体栈，保留技术专著感。
- 代码使用系统等宽字体栈。
- 正文默认 `16px`、行高 `1.7`；密集元数据不小于 `12px`。
- 主标题在桌面为 `30px` 左右，手机为 `24px`。
- 正文内容内部最大阅读宽度为 `920px`，超宽屏不无限拉长行宽。
- 页面放大至 200% 时不出现正文遮挡或必须双向滚动的布局。

## 4. 页面架构

页面由以下独立组件组成：

1. `SiteHeader`：站点名、当前固定版本、全站搜索入口和移动端菜单入口。
2. `ChapterNav`：学习路径、章节树、当前页定位和跨专题入口。
3. `Article`：面包屑、标题、摘要、状态标签、正文、源码块、调用链和上下页。
4. `EvidenceRail`：页内目录、证据等级、迁移提醒、源码索引和引用入口。
5. `SidebarToggle`：左右栏各自的展开/收起按钮。
6. `Drawer`：手机端承载章节树或证据栏的模态抽屉。

页面使用语义化 `header`、`nav`、`main`、`article`、`aside` 和 `footer`。正文、章节与证据在初始 HTML 中完整存在；JavaScript 只增强交互，不负责加载关键内容。

## 5. 桌面布局

### 5.1 展开状态

- 顶部栏高度约 `58px`。
- 左栏默认宽度 `240px`。
- 中央栏使用 `minmax(0, 1fr)`，内部正文最大宽度 `920px`。
- 右栏默认宽度 `245px`。
- 站点外壳在超宽屏居中，最大宽度约 `1720px`。
- 左右栏在长页面中保持视口内可用，独立滚动时不得截断键盘焦点。

### 5.2 收起状态

- 左栏和右栏互相独立，任何一侧均可单独收起。
- 收起后保留约 `40px` 的图标轨道，而不是完全消失。
- 左轨道保留首页、章节、源码索引、迁移映射等入口。
- 右轨道保留页内目录、证据、警示、引用等入口。
- 图标必须有可见悬停提示和键盘聚焦提示。
- 左栏快捷键为 `Alt+[`，右栏快捷键为 `Alt+]`。

## 6. 响应式行为

### 6.1 宽屏桌面：`≥1200px`

左右栏默认展开。用户可以进入左收起、右收起或双侧收起状态，中央正文随可用宽度扩展，但仍受最大行宽限制。

### 6.2 笔记本：`721–1199px`

左栏默认展开，右栏默认收起为图标轨道。点击右侧图标临时展开对应面板；左栏仍可手动收起以进入专注源码模式。

### 6.3 手机：`≤720px`

正文为单栏，顶部保留当前章节和搜索入口。左右内容不占据固定栏位，而是分别作为抽屉打开：

- 抽屉默认关闭，不能因为桌面端保存了“展开”状态而在手机加载时自动遮挡正文。
- `Esc`、遮罩点击和明确的关闭按钮均可关闭抽屉。
- 抽屉打开后焦点进入抽屉并被约束在其中，关闭后回到触发按钮。
- 代码块允许横向滚动，禁止压缩、换行破坏或裁切代码。

## 7. 状态数据流

JavaScript 只管理界面偏好：

1. 页面加载后读取当前视口档位。
2. 桌面和笔记本读取本地保存的左右栏偏好。
3. 没有保存值时使用当前断点默认值。
4. 用户切换后立即更新 `aria-expanded`、布局类和本地偏好。
5. 进入手机档位时关闭两个抽屉，但不覆盖桌面偏好。
6. 返回桌面档位时恢复已保存偏好。

本地存储键固定为：

- `q3a.sidebar.left`
- `q3a.sidebar.right`

允许值只有 `expanded` 和 `collapsed`。其他值视为损坏数据并恢复断点默认值。

## 8. 渐进增强与异常处理

- JavaScript 被禁用或执行失败：桌面显示完整三栏；窄屏按正常文档流依次呈现章节、正文和证据，所有内容仍可访问。
- `localStorage` 不可用：当前会话仍可切换侧栏，但不持久化；不得抛出阻断页面的异常。
- 保存状态损坏：删除无效值并恢复当前断点默认状态。
- 页面没有页内目录、证据或迁移提示：不渲染对应按钮和空容器。
- 标题或路径过长：允许合理换行或省略显示，但完整值必须通过标题属性或可访问名称获得。
- 深层章节树：当前路径自动展开，其余节点按需折叠；键盘可以访问所有层级。
- 超宽代码：代码区域横向滚动，并保留复制与固定源码链接入口。
- 打印或导出 PDF：移除站点导航和交互控件，保留标题、正文、证据状态、引用和源码链接文本。

## 9. 可访问性

- 侧栏按钮使用真实 `button`，提供 `aria-controls` 和实时 `aria-expanded`。
- 抽屉使用适当的对话框语义、焦点约束和可见关闭按钮。
- 所有交互可通过键盘完成，焦点样式不可被移除。
- 文字与背景满足 WCAG AA 对比度；正文目标至少 `4.5:1`。
- 状态不得只靠颜色表达，必须包含文字或图标加可访问名称。
- 页面结构保持唯一 `h1` 和连续标题层级。
- 动效尊重 `prefers-reduced-motion`，收缩动画可以完全关闭。

## 10. 搜索与内容边界

顶部搜索框在本规格中只定义外观、焦点和结果入口。全文索引生成、查询算法和结果页由后续知识站计划定义。模板不得把正文或证据依赖于搜索脚本。

源码块展示文件路径、固定 commit 行号、复制按钮和远程固定版本链接。复制失败时不影响代码阅读；远程链接失效时仍保留本地文件与符号定位。

## 11. 验收标准

正式模板实现必须通过以下检查：

1. Playwright 在 `1440×900`、`1024×768` 和 `390×844` 三个视口生成并比对基线截图。
2. 左右栏可以独立切换，按钮状态与实际布局一致。
3. 桌面和笔记本刷新后恢复用户偏好；手机初始加载不自动打开抽屉。
4. `Alt+[`、`Alt+]` 和 `Esc` 行为符合规格。
5. `localStorage` 禁用或包含非法值时页面仍可用并恢复安全默认值。
6. JavaScript 禁用时所有正文、章节、证据和引用仍可访问。
7. 键盘焦点顺序、抽屉焦点约束、关闭后焦点恢复和 `aria-expanded` 均正确。
8. 正文颜色对比度满足 WCAG AA，页面放大 200% 不遮挡关键内容。
9. 长标题、深层目录、超宽代码、空证据栏和无页内目录的样例正确显示。
10. 打印预览只保留内容、证据与引用，不打印导航轨道或抽屉控件。
11. 不产生远程运行时依赖；断网后可从本地静态文件完整阅读。

## 12. 已确认决策

- 主要设备：宽屏桌面与笔记本同时支持，手机可查阅。
- 基础布局：工程工作台三栏。
- 视觉风格：技术专著暖纸张配色。
- 代码区域：深色高对比代码块。
- 侧栏：左右独立收缩，收起后保留图标轨道。
- 默认状态：宽屏双栏展开，笔记本右栏收起，手机双抽屉关闭。
- 状态持久化：桌面与笔记本偏好保存在本地；手机不自动恢复为打开状态。
- 运行方式：离线静态页面，原生 JavaScript 渐进增强。
