# 学习路径三页试点设计

## 目标

将现有的静态证据站从单一的 31 页线性目录，试验性地改成可理解的三层学习入口。试点只增加：首页入口重构、迁移边界课、实施路线总览；既有 31 页、离线搜索与固定版本证据不删除。

## 信息架构

左侧章节导航必须静态呈现三组：`入门教程`、`源码深读`、`实施路线`。试点页属于入门或实施轨道；既有 31 页均归入源码深读。全局相邻页仍按唯一 `order` 保持可遍历，以避免破坏既有深链接与测试的阅读顺序。

新增页面各自给出：本页解决的问题、学习目标、前置知识、读完能判断什么，以及以明确链接表达的下一步。首页同时说明三层的读法。实施路线只定义环境、单卡、单机多卡、多机、性能、验收六个阶段的观察对象和验收边界，不表述任何硬件运行结论。

## 真实性边界

所有训练、性能、兼容性、分布式与 checkpoint reload 的运行时结论继续标为 `pending_hardware`，不增加未经固定源码或项目材料支持的事实性声明。新增教程文字仅作为教学组织，引用现有证据或清楚声明为实验计划。

## 不变量

页面由目录驱动生成；所有资源本地加载；无 JavaScript 时完整导航可读；搜索内联数据和搜索索引保持等价；固定版本证据链接必须对新增后的整站校验；侧栏可收缩行为不变。
