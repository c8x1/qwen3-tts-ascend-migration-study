SFT 数据预处理与 Collate

沿官方 12Hz Base 单说话人配方，拆解原始 JSONL、离线 codec materialization、Dataset item 与双轨 collate 张量。

学习目标

  • 区分原始 JSONL 与离线加入 audio_codes 后的训练 JSONL
  • 按固定源码解释 text ids、参考 mel、双轨输入、masks 与 codec-0 labels
  • 识别 CUDA 预处理默认值、bucketing 缺口与运行时 sharding 边界

前置知识

  • 熟悉 PyTorch 单卡张量与训练循环
已核验项目声明静态推断待真机验证

JSONL 字段契约

状态：已核验 公开 README 的原始行提供 audio/text/ref_audio；prepare_data.py 保留原字典字段并加入 audio_codes，写成新的 JSONL。audio 指向训练目标音频，text 是转写，ref_audio 是参考说话人音频；audio_codes 不是 Dataset 在线生成，而是离线阶段物化。简写契约：audio/text/ref_audio → audio/text/ref_audio/audio_codes。

  • 证据 TGT-DATA-001

预处理设备默认值

状态：已核验 prepare_data 的 argparse 目标默认值是 default="cuda:0"，并把 args.device 原样传给 tokenizer 的 device_map=args.device。这只是官方预处理脚本的 target default，不表示自动设备选择，也不构成 NPU 映射。

  • 证据 TGT-DATA-003

离线 audio_codes 数据流

预处理入口finetuning/prepare_data.pyprepare_data.main
→
批量编码finetuning/prepare_data.pytokenizer_12hz.encode
→
移回 CPU 并转列表finetuning/prepare_data.pycode.cpu().tolist
→
写入带 audio_codes 的 JSONLfinetuning/prepare_data.pyjson.dumps / f.writelines

状态：已核验 真实 offline flow 是读取全部 JSONL 行、以最多 32 条音频组成 batch、调用 tokenizer.encode、逐条执行 code.cpu().tolist 写回字典，最后序列化并写出 JSONL。最后不足 32 条的 batch 走同一 encode/materialize 路径。

  • 证据 TGT-DATA-001

Collate：双轨输入、masks 与 labels

→
双轨输入与 codec 张量finetuning/dataset.pyinput_ids / codec_ids
→

→
构造 codec-0 labelsfinetuning/dataset.pycodec_0_labels

状态：已核验 TTSDataset.collate_fn 分配双轨 input_ids=(B,T,2) 与 codec_ids=(B,T,16)。轨 0 放 text/control/pad，轨 1 放 codec control、speaker slot 与 codec-0；text_embedding_mask、codec_embedding_mask、codec_mask、attention_mask 控制各段，codec-0 labels 只在音频 codec-0 与 EOS 位置监督，其余为 -100。ref_mels 直接沿 batch 维拼接。

  • 证据 TGT-DATA-002

数据边界

状态：静态推断 固定脚本以 shuffle=True 构造普通 DataLoader，但本页不作 bucketing 声明。DataLoader 之后会交给 accelerator.prepare，因此 sharding 属于 Accelerate runtime 讨论；不能由 Dataset 文件静态推断多进程切分结果。
