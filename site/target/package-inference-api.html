包结构与推理 API

从发布包边界进入公开推理包装器，按固定源码顺序理解注册、装载与三类生成入口。

学习目标

  • 区分源码仓、sdist 与模型运行所需外部材料的边界
  • 沿固定源码路径解释 Qwen3TTSModel.from_pretrained 的注册与装载顺序
  • 定位 Base、VoiceDesign 与 CustomVoice 三类公开包装器接口

前置知识

  • 熟悉 PyTorch 单卡张量与自回归生成
已核验项目声明静态推断待真机验证

包入口、依赖与发布边界

状态：已核验 pyproject.toml 固定包元数据、Python 下限与运行依赖；qwen_tts/core/__init__.py 导出 12Hz 与 25Hz tokenizer 配置/模型类；根包导入模型与 tokenizer 包装器，但 __all__ 只列版本；python -m qwen_tts 会打印 CLI 入口提示。源码根的 Apache License 2.0 文本不自动覆盖权重、数据或其他材料。

状态：已核验 MANIFEST.in 对 sdist prune assets、examples 与 finetuning；这只描述源码分发包的裁剪规则，不等于源码仓中不存在 examples/finetuning。

目标项目 CLI 默认值

状态：已核验 官方目标 CLI 的默认设备、dtype 与注意力实现分别是 cuda:0、BF16 与 FlashAttention-2（FA2），解析后的值会传入 from_pretrained。这些是目标项目面向 NVIDIA/CUDA 的默认值描述，不是设备迁移建议，也不是任何替代后端的映射结论。

→

状态：已核验 create_voice_clone_prompt 接收参考音频、可选参考文本与 x_vector_only_mode；generate_voice_clone 接收待合成 text/language 以及 prompt 或参考输入，随后调用条件模型 generate，再把 audio_codes 交给 speech tokenizer decode。这里只记录公开包装器契约，不把外部音频、权重或执行结果视为已验证。

  • 证据 TGT-API-002

VoiceDesign 公开入口

自然语言风格指令qwen_tts/inference/qwen3_tts_model.pyQwen3TTSModel.generate_voice_design
→
条件模型生成qwen_tts/core/models/modeling_qwen3_tts.pyQwen3TTSForConditionalGeneration.generate
→
speech tokenizer 解码qwen_tts/inference/qwen3_tts_tokenizer.pyQwen3TTSTokenizer.decode

状态：已核验 generate_voice_design 的主要输入是 text、language 与描述目标声音/风格的 instruct；它不构造参考音频 prompt，而是把 instruct_ids 传给条件模型 generate，再解码 audio_codes。

  • 证据 TGT-API-003

CustomVoice 公开入口

预定义 speaker 入口qwen_tts/inference/qwen3_tts_model.pyQwen3TTSModel.generate_custom_voice
→
条件模型生成qwen_tts/core/models/modeling_qwen3_tts.pyQwen3TTSForConditionalGeneration.generate
→
speech tokenizer 解码qwen_tts/inference/qwen3_tts_tokenizer.pyQwen3TTSTokenizer.decode

状态：已核验 generate_custom_voice 的主要输入是 text、language、预定义 speaker 名称与可选 instruct；它先校验 speaker/language，再把 speakers 与 instruct_ids 传给条件模型 generate。它与以自然语言描述声音的 VoiceDesign 不是同一参数契约。

  • 证据 TGT-API-003
