复合模型架构与生成流

拆开复合配置、说话人编码器、Talker、残差码本预测器与目标源码中的 FP32 精度岛。

学习目标

  • 解释复合配置如何嵌套 speaker、Talker 与 code predictor
  • 区分 Talker 的 codec-0 输出与由 num_code_groups 决定的残差码本预测
  • 定位 RoPE、RMSNorm 与 eager attention 的 FP32 计算边界

前置知识

  • 熟悉 PyTorch 单卡张量与自回归生成

Speaker encoder：mel 到 embedding

状态：已核验 Qwen3TTSSpeakerEncoder 是 ECAPA 风格说话人编码器。配置类的 extraction contract 默认 sample_rate=24000、mel_dim=128，模型 forward 产出 speaker embedding；这里组合的是配置与静态张量契约，不声称音频已在本机执行。

  • 证据 TGT-MODEL-001
  • 证据 TGT-CONFIG-001

Talker：text+codec 到 codec-0

状态：已核验 Qwen3TTSTalkerForConditionalGeneration 把文本与已有 codec 条件送入 decoder，并由 LM head 预测 codec-0 logits。codec-0 是 Talker 的主序列输出，不能与后续残差码本组混为一谈。

  • 证据 TGT-MODEL-002

MTP：由配置决定的残差码本

状态：已核验 训练 helper forward_sub_talker_finetune 读取 Talker hidden state 与 codec_ids，构造子 Talker 输入后调用 code_predictor.forward_finetune，并返回 logits 与 residual-codebook loss；它不属于 runtime inference chain。

状态：静态推断 静态公式是 residual groups = num_code_groups - 1；配置类默认值为 32，但实际发布 checkpoint 配置 pending，当前固定 evidence 没有锁定其真实 num_code_groups，因此不把默认值换算成发布模型结论。

  • 证据 TGT-MODEL-004
  • 证据 TGT-MODEL-002

目标源码的 FP32 精度岛

状态：已核验 目标模型源码中，Talker RoPE 在关闭 autocast 后用 FP32 计算再恢复输入 dtype；RMSNorm 的方差用 FP32；eager attention 的 softmax 用 FP32 再转回 query dtype。这些是官方目标实现的局部精度范围，不是迁移映射。

  • 证据 TGT-MODEL-005
  • 证据 TGT-MODEL-006
  • 证据 TGT-MODEL-007
